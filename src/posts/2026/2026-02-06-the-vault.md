## VEX

### Camera cull

Camera cull in VEX. Point the `cam` parameter at your camera (operator path).

```vex
string cam = ch("cam");
float buf = ch("buf");
vector ndc = toNDC(chs("cam"), @P);
if (ndc.x < -buf || ndc.x > (1+buf) || ndc.y < -buf || ndc.y > (1+buf) || ndc.z > 0) {
    removepoint(geoself(), @ptnum);
}
```

### Looping value from 0–1

```vex
@timeloop = (@Time % 2) / 2;
```

### Attribute from other input

Read an attribute from another input (e.g. input 2, `@P`).

```vex
@P = point(1, "P", @ptnum);
```

### Point in centre of an object

Second input = geometry; use an Add SOP with one point. Wrangle runs on that point.

```vex
vector min, max;
getbbox(1, min, max);
vector centre = avg(min, max);
@P = centre;
```

### Using point() expression

In an expression, get position of a point. Second number is axis: 0 = X, 1 = Y, 2 = Z.

```
point("geometry_path", 0, "P", 0)  // X
point("geometry_path", 0, "P", 1)  // Y
point("geometry_path", 0, "P", 2)  // Z
```

### Delete points based on colour

Uses a spare parameter for `threshold`.

```vex
if (@Cd < ch("threshold")) {
    removepoint(0, @ptnum);
}
```

### Delete points with slider

Random delete with threshold and seed. Use `@id` instead of `@ptnum` if you have stable id attributes.

```vex
if (rand(@id * ch('seed')) < ch('threshold')) {
    removepoint(0, @ptnum);
}
```

### Group points with slider

```vex
if (rand(@ptnum) > ch('threshold')) {
    i@group_mygroup = 1;
}
```

### Rotate @orient around an axis

Rotate `@orient` around an axis. `set(0,1,0)` is the axis (here, Y). Use a spare `rot` parameter (radians).

```vex
vector4 rot = quaternion(set(0, 1, 0) * ch("rot"));
@orient = qmultiply(p@orient, rot);
```

## Expressions

### Automatic filepath

Output path that stays organized. Uses a `ver` integer parameter so you can bump version and keep old caches.

```
${JOB}/geo/${OS}/v`padzero(3, chsop("ver"))`/${OS}.${F}.bgeo.sc
```

### Fit expression in Carve SOP

Animate carve along a curve over time (e.g. in First U or similar).

```
fit($F, $FSTART, $FEND, 0, 1)
```

General form: `fit(num, oldmin, oldmax, newmin, newmax)`.

### Get output from opinputpath

Resolve a connected input: `0` = first input, `1` = second, etc.

```
`opinputpath("..", 0)`
```

### bbox and centroid

For pyro/smoke container: use **Size** and **Center** with bbox and centroid. Axis: bbox uses 6=X, 7=Y, 8=Z; centroid uses 0=X, 1=Y, 2=Z.

```
bbox("inputPathToBounding", 6)
centroid("inputPathToBounding", 0)
```

## Tips

### Connecting detached curves

To fuse splines into one primitive, use the **PolyPath** SOP (spline fuse). It turns edges into polygon curves that stop at branch points, deduplicates edges, and can clean up topology. Handy for merging detached curve segments.

### HDA namespace and versioning

Use a namespace and version so Houdini loads the right asset and you don’t overwrite older instances.

- **Operator name:** `AFX::renderkit::1.0`
- **Operator label:** `renderkit`

Keeps versioning and avoids missing HDAs when opening old scenes. [SideFX: namespaces](https://www.sidefx.com/docs/houdini/assets/namespaces).
